# docs/ ディレクトリ構成

```
docs/
├── README.md              ← このファイル
├── features/              機能仕様
│   └── 機能概要.md         全機能の概要（F01-F09）、画面一覧（S01-S12）
├── plans/                 開発計画
│   └── 開発計画.md         フェーズ別の実装計画、DB設計
├── architecture/          設計・技術ドキュメント
│   └── （DB設計図、API設計、シーケンス図等を配置）
├── rules/                 開発ルール
│   └── 開発ルール.md       TDD、セキュリティ、コミットルール、コーディング規約
└── research/              技術調査・検討メモ
    └── （API選定、ライブラリ比較等の調査結果を配置）
```

## 各ディレクトリの役割

| ディレクトリ | 内容 | 更新タイミング |
|------------|------|--------------|
| `features/` | 機能仕様・画面仕様 | 要件変更時 |
| `plans/` | 開発計画・フェーズ管理 | フェーズ進行時 |
| `architecture/` | DB設計、API設計、シーケンス図 | 設計変更時 |
| `rules/` | 開発ルール・規約 | ルール追加・変更時 |
| `research/` | 技術調査・検討メモ | 調査実施時 |
